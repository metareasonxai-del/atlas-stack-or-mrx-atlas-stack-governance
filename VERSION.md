# Versioning (Founder‑Grade)

Use founder‑grade versioning to mark milestone progress across doctrine, agents, pipelines, UDP, keys, and culture layers.

- v0.1.0 — Doctrine Alpha
- v0.2.0 — Agents + Pipelines
- v0.3.0 — UDP
- v0.4.0 — Keys
- v0.5.0 — Ecosystem Map
- v0.6.0 — Culture Layer
- v1.0.0 — Public Release

Release process

- Tag releases using the VERSION.md milestone and update changelogs describing doctrine, spec, and activation changes.
- For major doctrine or governance changes, record the proposal, discussion, and consensus alongside the release notes.
